# Kotlinでの開発について

## コード全般

- 関数型プログラミングのスタイルに則り、説明的で宣言的なコードを書く
- 理由がない限り、イミュータブルなコードを書く
    - varではなくvalを使う
    - イミュータブルなコレクションを使う

## KDoc

- 作成した関数にはKDocで説明を記載する
- 特にpublicな関数には必ず説明を記載する
- KDocは日本語で記載する
- KDocで引数や戻り値に言及する場合、`@param`や`@return`を使うのではなく、[]で仮引数名や型を囲んで参照する

良い例:

```kotlin
/**
 * [task]をスレッドプール上で非同期実行する。[timeout]を超えた場合はタイムアウト例外が発生する。
 */
fun executeTask(task: Task, timeout: Duration)
```

悪い例:

```kotlin
/**
 * タスクをスレッドプール上で非同期実行する。タイムアウト時間を超えた場合はタイムアウト例外が発生する。
 *
 * @param task 実行するタスク
 * @param timeout タイムアウト時間
 */
fun executeTask(task: Task, timeout: Duration)
```

## クラス

- そのクラスの責務が何であるかを常に意識して関数やプロパティを持たせる
- クラスが複数の責務を持ち始めたり、責務が肥大化し始めたりした場合には、別のクラスに処理を渡せないか、あるいは新しいクラスに分割できないかを検討する
